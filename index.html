Operators

Common categories: arithmetic, assignment, comparison, logical, bitwise, ternary, instanceof.

Examples

// arithmetic
int a = 5 + 3 * 2;   // 11

// assignment and compound
a += 4;             // a = a + 4

// comparison
boolean b = (a > 10);

// logical
boolean c = (a > 5) && (b);

// ternary
int x = (a > 10) ? 1 : 0;

// instanceof
String s = "hi";
boolean isStr = s instanceof String;

Control Flow

If / Else

if (score >= 90) {
    System.out.println("A");
} else if (score >= 75) {
    System.out.println("B");
} else {
    System.out.println("C");
}

Switch (modern cases available since Java 14+)

int day = 3;
switch (day) {
    case 1 -> System.out.println("Mon");
    case 2 -> System.out.println("Tue");
    case 3 -> System.out.println("Wed");
    default -> System.out.println("Other");
}

Classes & Objects — Basics

Classes are blueprints for objects. Fields (state) + methods (behavior).

Simple Class Example

public class Person {
    // fields
    private String name;
    private int age;

    // constructor
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // getter
    public String getName() { return name; }

    // setter
    public void setName(String name) { this.name = name; }

    // method
    public void sayHi() {
        System.out.println("Hi, I'm " + name);
    }
}

// usage
Person p = new Person("Asha", 20);
p.sayHi();   // Hi, I'm Asha

OOP Concepts

Four pillars: Encapsulation, Inheritance, Polymorphism, Abstraction.

Encapsulation

Keep fields private, expose behavior through methods (getters/setters). Protects invariants.

public class BankAccount {
    private double balance;

    public BankAccount(double b) { this.balance = b; }
    public double getBalance() { return balance; }
    public void deposit(double amt) {
        if (amt > 0) balance += amt;
    }
}

Inheritance

One class extends another: re-use and subtype relationship.

public class Animal {
    public void speak() { System.out.println("..."); }
}

public class Dog extends Animal {
    @Override
    public void speak() { System.out.println("Woof"); }
}

Polymorphism

Same reference type, multiple implementations. Method overriding is run-time polymorphism.

Animal a = new Dog();
a.speak();  // "Woof" — Dog's override runs

Abstraction

Hide complex implementation details. Achieved via abstract classes and interfaces.

public interface Shape {
    double area();
}

public class Circle implements Shape {
    private double r;
    public Circle(double r) { this.r = r; }
    public double area() { return Math.PI * r * r; }
}

Constructors & this / super

public class A {
    public A() { System.out.println("A()"); }
}
public class B extends A {
    public B() {
        super();      // calls A()
        System.out.println("B()");
    }
}

Method Overriding vs Overloading

Overriding — subclass changes behavior of superclass method (same signature). Overloading — same method name different parameters.

final, static, abstract

  • final — prevents modification (final class, method or variable).
  • static — class-level member shared by all instances.
  • abstract — class/method must be implemented by subclass.

Example: Small OOP design

public abstract class Vehicle {
    private String model;
    public Vehicle(String m) { model = m; }
    public String getModel() { return model; }
    public abstract void drive();
}

public class Car extends Vehicle {
    public Car(String m) { super(m); }
    public void drive() { System.out.println("Car " + getModel() + " driving"); }
}

Extras — Helpful Notes

Exception Handling

try {
    // risky
} catch (IOException e) {
    e.printStackTrace();
} finally {
    // always
}

Interfaces vs Abstract Classes

Interface: multiple inheritance of type, default/static methods allowed. Abstract class: partial implementation, can hold state.

Packages

Organize classes: package com.example.app; Use imports to access other packages.

Common Collections

ArrayList, LinkedList, HashSet, HashMap — part of java.util. Use generics: List<String>.
